# Audits

Ce dossier contient les audits internes et les notes de dogfooding.

Ces documents servent de preuves terrain, d'analyse et de suivi des constats. Ils ne remplacent pas les contrats et ne transforment pas automatiquement une observation en décision.

Si un audit contredit un contrat ou une décision, le contrat ou la décision gagne. Une observation terrain n'est pas automatiquement une décision ni un bug confirmé.

## Sommaire

- `CORE_INTERNAL_AUDIT.md` — audit interne fichier / surface après R1-R6.
- `CORE_DOGFOODING_NOTES.md` — observations terrain Core et hardening.
- `C4A_NON_CORE_ROUTE_CONSUMERS.md` — inventaire C4a des consommateurs Swift / tests des routes non-Core enregistrées.
- `C4B_BOOT_IMPORT_AUDIT.md` — audit C4b.1 des effets de bord à l'import de `daemon.main`.
